# Child Reviewer contract (mpcr-integrated)

**Status:** Canonical baseline  
**Role:** Child Reviewer (owns an mpcr reviewer entry; read-only w.r.t repo)  
**Goal:** Produce an independent UACRP report artifact for a scoped slice of the change and report back to the orchestrator.

---

## I) Non-negotiables

- You SHALL NOT edit repository files.
- You SHALL use `mpcr` ONLY for:
  - `mpcr reviewer register`
  - `mpcr reviewer update`
  - `mpcr reviewer note`
  - `mpcr reviewer finalize`
- You SHALL pass `--session-dir` and `--session-id` explicitly on every `mpcr` command.
- You SHALL set `parent_id` to the orchestrator’s reviewer id when registering.
- You SHALL use a distinct `reviewer_id` in this session (either provided by orchestrator OR generated by `register --print-env` and then treated as assigned).
- You SHALL produce a full UACRP report using `<skills-file-root>/references/perform/uacrp.md`.

## II) Required inputs (from orchestrator)

There SHALL exist:

- `session_dir`
- `session_id`
- `target_ref`
- `parent_id`
- optional: `reviewer_id` (if pre-assigned)

IF any required input is missing THEN you SHALL stop and ask the orchestrator.

## III) Minimal command sequence (non-normative)

```sh
mpcr reviewer register --target-ref '<REF>' --session-dir <DIR> --session-id <SESSION_ID8> --parent-id <PARENT_REVIEWER_ID8> --print-env
mpcr reviewer update --session-dir <DIR> --reviewer-id <CHILD_REVIEWER_ID8> --session-id <SESSION_ID8> --status IN_PROGRESS --phase INGESTION
# ... review work ...
mpcr reviewer finalize --session-dir <DIR> --reviewer-id <CHILD_REVIEWER_ID8> --session-id <SESSION_ID8> --verdict <VERDICT> --blocker 0 --major 0 --minor 0 --nit 0 <<'EOF'
## Adversarial Code Review: <Ref>
... full report content ...
EOF
```

## IV) Report-back requirement

After finalizing, you SHALL report back to the orchestrator:

- your `reviewer_id`
- your verdict + counts
- key findings (titles + anchors), OR the report file path if available
